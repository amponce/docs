## Overview

The call history API lets you list, filter, and inspect past and in-progress calls. Every call record includes metadata, status, timing information, and links to detailed transcripts and cost breakdowns.

## List Calls

Retrieve calls with pagination and optional filters:

<Tabs items={["cURL", "TypeScript", "Python"]}>
<Tab value="cURL">
```bash
curl "https://your-host/v1/calls?limit=20&offset=0&status=completed&direction=outbound" \
  -H "Authorization: Bearer ra_your_api_key"
```
</Tab>
<Tab value="TypeScript">
```typescript
const params = new URLSearchParams({
  limit: "20",
  offset: "0",
  status: "completed",
  direction: "outbound",
});

const response = await fetch(`https://your-host/v1/calls?${params}`, {
  headers: { "Authorization": "Bearer ra_your_api_key" },
});

const { data, hasMore } = await response.json();
```
</Tab>
<Tab value="Python">
```python
import requests

response = requests.get(
    "https://your-host/v1/calls",
    headers={"Authorization": "Bearer ra_your_api_key"},
    params={
        "limit": 20,
        "offset": 0,
        "status": "completed",
        "direction": "outbound",
    },
)

result = response.json()
calls = result["data"]
has_more = result["hasMore"]
```
</Tab>
</Tabs>

### Query Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `limit` | `number` | `50` | Results per page (1-500). |
| `offset` | `number` | `0` | Number of records to skip. |
| `status` | `string` | - | Filter by status: `queued`, `ringing`, `in-progress`, `completed`, `failed`. |
| `direction` | `string` | - | Filter by direction: `inbound` or `outbound`. |

### Response

{
  "data": [
    {
      "id": "call_xYzAbCdEf123456789012",
      "orgId": "org_abc123",
      "agentId": "agent_abc123def456",

### Pagination

Use `limit` and `offset` for cursor-based pagination:

```typescript
let offset = 0;
const limit = 50;
let allCalls = [];

while (true) {
  const response = await fetch(
    `https://your-host/v1/calls?limit=${limit}&offset=${offset}`,
    { headers: { "Authorization": "Bearer ra_your_api_key" } }
  );
  const { data, hasMore } = await response.json();
  allCalls.push(...data);

  if (!hasMore) break;
  offset += limit;
}
```

## Get Call Details

Retrieve a single call with full details:

```bash
curl "https://your-host/v1/calls/{callId}" \
  -H "Authorization: Bearer ra_your_api_key"
```

## Call States

| Status | Description |
|---|---|
| `queued` | Call record created, awaiting initiation. |
| `ringing` | Outbound call is ringing the destination. |
| `in-progress` | Call is connected and audio is streaming. |
| `completed` | Call ended normally. |
| `failed` | Call failed to connect or encountered an error. |

## End Reasons

| End Reason | Description |
|---|---|
| `agent_hangup` | The AI agent ended the call (e.g. conversation complete). |
| `caller_hangup` | The human caller hung up. |
| `terminated` | The call was terminated via the API (`POST /calls/:id/terminate`). |
| `initiation_failed` | Outbound call failed to initiate (e.g. invalid number). |
| `error` | An unexpected error occurred during the call. |
| `timeout` | The call exceeded the maximum duration. |

## Retrieve Call Events

For debugging, use the events endpoint to get the full event buffer from a call:

```bash
curl "https://your-host/v1/calls/{callId}/events?type=stt_final,llm_complete&limit=100" \
  -H "Authorization: Bearer ra_your_api_key"
```

### Query Parameters

| Parameter | Type | Description |
|---|---|---|
| `type` | `string` | Comma-separated event types to filter (e.g. `stt_final,latency`). |
| `limit` | `number` | Maximum events to return (from most recent). |
| `since` | `string` | ISO 8601 timestamp. Only return events after this time. |

### Response

```json
{
  "callId": "call_xYzAbCdEf123456789012",
  "count": 42,
  "events": [
    {
      "timestamp": "2026-02-08T12:00:05.123Z",
      "callId": "call_xYzAbCdEf123456789012",
      "type": "stt_final",
      "data": {
        "text": "Hi, I need help with my account.",
        "confidence": 0.97
      }
    },
    {
      "timestamp": "2026-02-08T12:00:06.456Z",
      "callId": "call_xYzAbCdEf123456789012",
      "type": "llm_complete",
      "data": {
        "text": "Of course! I'd be happy to help you with your account. Could you tell me your account number?",
        "model": "gpt-4o",
        "tokensUsed": 42
      }
    }
  ]
}
```

<Callout type="info" title="Tip">
  If the call is still active, events are served from the in-memory circular buffer (up to 1,000 events). After the call ends and the inspector is disposed, the endpoint returns an empty array. Use the [DevTools WebSocket](/monitor/devtools) for live streaming during active calls.
</Callout>
